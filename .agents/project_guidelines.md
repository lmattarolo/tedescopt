# Project Guidelines - German-Italian Vocabulary Trainer (TedescoPT)

Welcome to the German-Italian Vocabulary Trainer project! This document outlines the project scope, architecture, design decisions, and coding standards. Adhere to these guidelines when modifying or extending this codebase.

---

## Project Overview

A self-hosted, responsive web tool that helps users learn German vocabulary from Italian, specifically designed to ingest glossary lists from Zanichelli German textbooks (e.g., unit glossaries).

### Key Features
1. **Interactive Flashcards**:
   - **Front**: Italian word.
   - **Back (Step 1)**: German translation and gender indicator (if it's a noun: *der/die/das* or *m/f/n*).
   - **Back (Step 2 - Optional)**: German plural form, if it exists, guessable by flipping the card a second time before answering.
2. **Spaced Repetition / Weighted Selection**:
   - A semi-random selection algorithm.
   - Cards guessed incorrectly are shown more frequently (higher weight).
   - Cards guessed correctly are shown less frequently (lower weight).
   - No card is completely starved; all cards have a non-zero probability of appearing.
3. **Data Ingestion via AI Assistant**:
   - Users drop glossary photos (like `einheit_1.jpg`) into the `zanichelli_vocabulary` folder.
   - The AI Assistant parses the photo and outputs a structured `einheit_1.json` file in the workspace.
   - The web app allows uploading/importing these JSON files directly to add vocabulary units.
4. **Mobile & PC Friendly**:
   - Mobile-first, responsive, premium dark-themed UI.
   - Configured as a Progressive Web App (PWA) so it can be installed on smartphone home screens.
5. **Hosting & Synchronization**:
   - Hosted for free on GitHub Pages.
   - Progress is stored locally in the browser (LocalStorage).
   - Backups (vocabulary + stats) can be exported and imported as JSON to sync progress between devices (PC and smartphone).
6. **Deck & Word Editor**:
   - A management interface to view, edit, add, or delete individual words to easily fix any OCR typos.

---

## Tech Stack & Architecture

- **Frontend Framework**: React (Vite SPA) using JavaScript.
- **Deployment**: GitHub Pages (using a GitHub Actions workflow for automatic deployment).
- **Styling**: Vanilla CSS with modern custom properties, dark-mode premium theme, glassmorphism, and smooth animations.
- **State & Storage**: Client-side LocalStorage.
- **Backend**: None (pure client-side).

---

## Key Decisions & Rationale

1. **Client-Side Only Web App (Vite SPA)**:
   - *Rationale*: Simplifies deployment, zero cloud database costs, runs entirely in the browser.
2. **GitHub Pages Hosting**:
   - *Rationale*: Free, highly reliable, and accessible on smartphones at any time without needing the PC to be turned on.
3. **File-Based Import/Export for Sync**:
   - *Rationale*: A simple export-to-JSON and import-from-JSON workflow allows syncing progress without needing a complex backend sync service.
4. **AI-Assisted Ingestion**:
   - *Rationale*: The AI Assistant (Antigravity) will parse photos directly into JSON, eliminating the need to write and configure local OCR scripts or manage API keys.
5. **LocalStorage for Persistence**:
   - *Rationale*: With 12 units (~500-1000 words + study stats), the total data size is under 1MB. LocalStorage is simple, fast, and highly reliable for this scale.

---

## Style & Development Guidelines

- **Mobile First**: All UI components must look spectacular on mobile screens (portrait) as well as desktop.
- **Theme**: Premium dark aesthetic with vibrant accents (e.g., deep charcoal background, glassmorphism card panels, smooth scale and rotate transitions for card flips).
- **No Placeholders**: Ensure all components and data loaders are fully functional.

## Ingestion Schema (JSON)

Vocabulary files generated by the OCR script should follow this JSON format:

```json
{
  "unit": "Einheit 1",
  "words": [
    {
      "id": "unique-uuid-or-hash",
      "italian": "la sedia",
      "german": "Stuhl",
      "gender": "der",
      "plural": "Stühle",
      "partOfSpeech": "noun"
    },
    {
      "id": "another-uuid",
      "italian": "studiare",
      "german": "studieren",
      "gender": null,
      "plural": null,
      "partOfSpeech": "verb"
    }
  ]
}
```
